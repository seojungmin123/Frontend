# 변수

변하는 값을 저장하는 저장공간(메모리). 임시 저장 장소.

const a = 5; < 변수 a는 5와 같다. (대입, 치환)

**유의사항**

- 영문자와 숫자, $와 _만 사용.
- 대소문자 구별함.
- 공백 대신 다음 단어의 첫문자를 대문자로 씀. < my name X myName O (camelCase)
- 첫글자는 숫자X
- 예약어 사용X
- 가급적 상수는 대문자로 알려줄 것.
- 변수명은 읽기 쉽고 이해할 수 있게, 의미에 맞게 선언.

<br>

---

**const** < 값이 바뀔 수 없음. 재선언, 재할당 할 수 없음. (**상수**임. + 대문자 상수는 ‘하드 코딩한’ 값의 별칭을 만들 때 주로 사용. 실행 전에 이미 값을 알고 있고, 코드에서 직접 그 값을 쓰는 경우에 사용한다.)

**let** < 값이 바뀔 수 있음. 재선언 할 수 없으나 재할당 할 수 있음. ( let은 새로운 것을 생성할 때 사용.)

```js

const A = 5; // 상수 (변하지않음)
const B = 2;
let myName = "seo"; // 새로운 let 변수 생성 (let 사용)

console.log(A + B);  // 7
console.log("hello" + myName);  // hello yujin

myName = "seo"; // let 변수 업데이트 (let 생략)

console.log("your new name is " + myName);  // your new name is seo

//대문자상수
const birthday = '18.04.1982';

const age = someCode(birthday);
// 이 경우엔 이미 값을 알고있는 BIRTHDAY가 대문자상수로 바꾸기에 적합함.
// 반면 age는 런타임에 평가되고 실행될때마다 값이 바뀔 수 있음. 대문자상수에 적합하지않음.
```

보통 const를 기본으로 사용하고, 변수를 업데이트(재할당) 하고싶다면 let을 사용.

- const, let이 생기기 전에는 var를 썼었음. (어디서든 값 업데이트 가능. 현재 사용X)
    
    그러나 문제가 좀 많음
    
    1. 호이스팅시 변수의 선언과 초기화(undefined)를 같이 시켜버리는 문제가 있었음.
    
    ```js
    console.log(a); // undefined
    var a = 1; 
    console.log(a); // 1
    ```
    
    1. 전역변수와 지역변수의 차이가 애매했음.
    
    (함수만 지역변수로 호이스팅되고, 나머지는 다 전역변수로 올려버림..)
    
    ```js
    for(var i = 1; i<5; i++){
    	console.log(i); // 1, 2, 3, 4
    }
    console.log(i); // 5
    ```
    
    1. 이름 중복 가능(????)
    
    ```js
    var a = 1;
    console.log(a); // 1
    var a = 2;
    console.log(a); // 2
    ```
    
    2015년에 ES6로 업그레이드 되면서 let이 나옴.
    
    2, 3번은 해결되었고
    
    1번은 let도 호이스팅은 됨. 그러나 Temporal Death Zone(TDZ), 일시적 사각지대를 만들어 선언문이 나오기 전까지는 그 변수에 접근할 수 없도록 했음. (에러 띄워줌)
    
    ```js
    console.log(a); // Error : Cannot access 'a' before...
    let a = 1; 
    console.log(a);
    ```
    

지역변수 : 함수 내부에서만 사용.

전역변수 : 어디서든 접근 가능.
